     kubectl get deployment {{ include "open-appsec.webhook.fullname" . }} \

## Enable injection on an existing deployment

The webhook only mutates pods that match BOTH `namespaceSelector` AND
`objectSelector`. To opt a workload in:

  # 1. Label the workload's namespace
  kubectl label namespace <target-ns> \
{{- range $key, $value := .Values.agent.namespaceSelector.matchLabels }}
    {{ $key }}={{ $value }} --overwrite{{- end }}

  # 2. Label the Deployment / StatefulSet pods (label goes on .spec.template.metadata.labels)
  #    For the default selector this is the Istio gateway:
  kubectl patch deployment <gateway-deployment> --namespace <target-ns> \
    --type=strategic --patch '{"spec":{"template":{"metadata":{"labels":{
{{- range $key, $value := .Values.agent.objectSelector.matchLabels }}
      "{{ $key }}":"{{ $value }}"
{{- end }} }}}}}'

  # 3. Restart so pods pass through admission and get the sidecar injected
  kubectl rollout restart deployment <gateway-deployment> \
    --namespace <target-ns>

Verify injection landed by counting containers (expect 2 — gateway + agent):

  kubectl get pod -l <gateway-label-selector> --namespace <target-ns> \
    -o jsonpath='{.items[*].spec.containers[*].name}'

## Troubleshooting

- **Pods stuck in `ContainerCreating` with `MountVolume.SetUp failed`**:
  agent.persistence is enabled but no PV is available. Either set
  `agent.persistence.config.storageClass` / `agent.persistence.data.storageClass`
  to a class that has a provisioner, or supply
  `agent.persistence.{config,data}.existingClaim` pointing at pre-created PVCs.

- **Admission timeouts** (`webhook ... failed: context deadline exceeded`):
  the webhook pod may be slow to start. Bump `webhook.timeoutSeconds`
  (currently `{{ .Values.webhook.timeoutSeconds }}`) or
  `webhook.replicaCount` for redundancy.

- **`x509: certificate signed by unknown authority`**: the webhook's
  `caBundle` patch hasn't landed yet, or RBAC on the webhook SA is too
  narrow. Confirm the webhook pod's logs include `==> Key material
  generated` and the ClusterRoleBinding `{{ include "open-appsec.webhook.fullname" . }}:mutate-injection`
  grants `mutatingwebhookconfigurations`.

- **Tuning Deployment is in `CrashLoopBackOff`**: `appsec.tuning.enabled=true`
  but no database is reachable. Either set `postgresql.enabled=true` for
  the bundled subchart, or configure
  `appsec.tuning.externalDatabase.host` / `existingSecret`.
